# C ABI

Build with `--features capi` (and `fbink` on device if you need the real
renderer from Rust; the current C app handle uses the mock renderer for frame
submit / tap dispatch — see limitations below).

The committed header is [`include/signalkit.h`](../include/signalkit.h)
(generated by cbindgen; do not edit by hand). Regenerate after FFI changes:

```sh
cbindgen --config signalkit/cbindgen.toml \
  --crate signalkit \
  --output signalkit/include/signalkit.h
```

(Exact invocation may match CI; keep the header committed.)

## Conventions

- **Opaque handles** — `sk_*_new` / matching `sk_*_free`.
- **Thread** — all calls on the thread that created `sk_app`.
- **Strings** — UTF-8 `const char*`, copied on the way in.
- **Errors** — failing calls set a thread-local message; read with
  `sk_last_error()` (do not free; valid until the next failing call).
- **Panics** — every entry point uses `catch_unwind`; a Rust panic becomes an
  error / no-op instead of unwinding into foreign frames. Release builds keep
  `panic = "unwind"` for that reason.

## Surface (v1)

| Area | Functions |
|------|-----------|
| Version / errors | `sk_version`, `sk_last_error` |
| `i64` signal | `sk_signal_i64_new/get/set/free` |
| Label | `sk_label_new`, `sk_label_set_text`, `sk_label_bind_i64`, `sk_label_free` |
| Button | `sk_button_new`, `sk_button_on_tap`, `sk_button_free` |
| Stack | `sk_vstack_new`, `sk_hstack_new`, `sk_stack_push_label/button`, `sk_stack_free` |
| App | `sk_app_new`, `sk_app_render_frame`, `sk_app_tap_at`, `sk_app_free` |

`slot`, `for_each`, string signals, and custom components are **not** exposed
yet.

## Minimal C sketch

```c
#include "signalkit.h"
#include <stdio.h>

static void on_inc(void *user) {
    sk_signal_i64 *sig = user;
    sk_signal_i64_set(sig, sk_signal_i64_get(sig) + 1);
}

int main(void) {
    sk_signal_i64 *count = sk_signal_i64_new(0);
    sk_label *label = sk_label_new("");
    sk_label_bind_i64(label, count, "Count: ", "");

    sk_button *inc = sk_button_new(" + ");
    sk_button_on_tap(inc, on_inc, count);

    sk_stack *root = sk_vstack_new(16);
    sk_stack_push_label(root, label);
    sk_stack_push_button(root, inc);

    sk_app *app = sk_app_new(root, 600, 800); /* consumes root */
    if (!app) {
        fprintf(stderr, "%s\n", sk_last_error());
        return 1;
    }

    sk_app_render_frame(app);
    /* Drive taps yourself, or integrate with your own input loop: */
    sk_app_tap_at(app, 100, 200);
    sk_app_render_frame(app);

    sk_app_free(app);
    sk_button_free(inc);
    sk_label_free(label);
    sk_signal_i64_free(count);
    return 0;
}
```

Link against `libsignalkit.so` and ensure the header is on the include path.
The KPM package `com.bd452.signalkit` installs the `.so` and header for
on-device consumers.

## Ownership notes

- `sk_app_new` **consumes** the root stack (do not `sk_stack_free` it afterward).
- Widgets pushed onto a stack are referenced by the tree; still free the
  handles you created when the app is done, matching the header’s create/free
  pairs (see `ffi.rs` for exact ownership if in doubt).
- Tap callbacks receive the `user_data` pointer you passed to
  `sk_button_on_tap`; the library does not free it.

## Limitations

- No on-device `sk_app_run` yet — C callers drive `render_frame` / `tap_at`
  (or call into a Rust wrapper that owns `FbinkRenderer` + `App::run`).
- Structural UI (`slot` / `for_each`) remains Rust-only.
- Prefer the Rust API for new Kindle apps; use the C ABI when embedding from
  another language.
