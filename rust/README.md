# SignalKit for Kindle (Rust)

A Rust port of SignalKit — a reactive UI framework — targeting Kindle
e-readers. The reactive model (`Signal`, `Component`, `Node`, `vstack` /
`hstack`, `slot`, `for_each`) is faithful to the original; because there is no
UIKit underneath, the port adds a retained widget tree, a stack layout solver,
an FBInk-based e-ink renderer, and a touch event loop.

**Library docs:** [`signalkit/README.md`](signalkit/README.md) and
[`signalkit/docs/`](signalkit/docs/).

## Workspace

| Crate | What it is |
|-------|------------|
| [`signalkit`](signalkit/) | The library. `rlib` for Rust consumers, `cdylib` + C ABI (feature `capi`) for other languages. FBInk backend behind feature `fbink`. |
| [`fbink-sys`](fbink-sys/) | FFI to statically-linked FBInk, generated with bindgen from the vendored `apps/com.bd452.fbink/vendor/FBInk` header. Built only for the ARM cross targets. |
| [`signalkit-demo`](signalkit-demo/) | Demo app (counter, keyed list, touch buttons). Device build under feature `fbink`; a host stub otherwise. |

## Building

**Full cold-start guide** (host vs cross, macOS/Linux/Docker, deploy,
troubleshooting): **[signalkit/docs/building.md](signalkit/docs/building.md)**.

**Host development / tests** (any OS; no cross toolchain):

```sh
cargo test -p signalkit                 # core, layout, structural, input, app
cargo test -p signalkit --features capi # + the C ABI
cargo run  -p signalkit-demo            # host stub: mounts + renders one frame
```

**On-device** cross builds need Linux x86_64 + koxtoolchain. On macOS use the
repo Docker image via [`scripts/build-in-container.sh`](../scripts/build-in-container.sh)
(see [building.md](signalkit/docs/building.md) §5):

```sh
docker build --platform linux/amd64 -t kinstaller-build .   # once
./scripts/build-in-container.sh apps/com.bd452.signalkitdemo/build.sh
```

- `apps/com.bd452.signalkit/build.sh` → `libsignalkit.so` + `signalkit.h`
- `apps/com.bd452.signalkitdemo/build.sh` → demo binary ([demo README](../apps/com.bd452.signalkitdemo/README.md))
- `./build.sh` at the repo root → full repo packages

Targets: `armv7-unknown-linux-gnueabihf` (kindlehf),
`armv7-unknown-linux-gnueabi` (kindlepw2). Init the FBInk submodule first
(`git submodule update --init --recursive`).

## C ABI

`signalkit/include/signalkit.h` (generated by cbindgen, committed) exposes
signals, labels, buttons, stacks, and the app loop. See the header and
[signalkit/docs/c-abi.md](signalkit/docs/c-abi.md). All calls must occur on the
thread that created the `sk_app`.
